StoryFox generates beautiful illustrations for every page of your storybook. The illustration system supports multiple AI providers, handles content safety challenges, and includes sophisticated retry logic to ensure every page gets an image.

## Generation architecture

Illustrations are generated concurrently with controlled parallelism:

<Steps>
  <Step title="Cover image first">
    The cover is generated first and can optionally be used as a character reference for remaining pages
  </Step>
  
  <Step title="Parallel page generation">
    Story pages generate concurrently with a configurable limit (default: 3 concurrent images)
  </Step>
  
  <Step title="Recovery passes">
    Failed pages are retried up to 3 times with progressively safer prompt variants
  </Step>
  
  <Step title="Final cover rescue">
    If the cover failed, a dedicated rescue attempt uses the safest prompts
  </Step>
</Steps>

```swift
let semaphore = AsyncSemaphore(limit: GenerationConfig.maxConcurrentImages)

try await withThrowingTaskGroup(of: (Int, String, CGImage?).self) { group in
    for page in pages {
        group.addTask {
            await semaphore.wait()
            defer { Task { await semaphore.signal() } }
            
            let image = try await self.generateSingleImage(
                prompt: enrichedPrompt,
                style: style,
                format: format
            )
            return (page.pageNumber, enrichedPrompt, image)
        }
    }
}
```

## Progressive prompt fallback

When image generation fails (usually due to content safety filters), StoryFox automatically tries progressively safer prompt variants:

### Variant chain for ImagePlayground

1. **Sanitized** - Full sanitized scene (~180 characters)
2. **LLM rewritten** - Foundation Model rewrites the prompt for safety
3. **Shortened** - First 18 keywords only
4. **High reliability** - First 12 keywords
5. **Fallback** - First 8 keywords + scene framing
6. **Ultra safe** - First 4 keywords + "sunny day"

```swift
private let variantLabels = [
    "sanitized",
    "llmRewritten",
    "shortened",
    "highReliability",
    "fallback",
    "ultraSafe"
]
```

### Cloud provider fast path

Cloud providers (OpenAI, Replicate, Hugging Face) skip the variant chain and use the full prompt directly:

```swift
if currentSettings.imageProvider.isCloud {
    let sanitizedPrompt = ContentSafetyPolicy.safeIllustrationPrompt(
        prompt,
        maxLength: 1000
    )
    let outcome = try await router.generateImage(
        prompt: sanitizedPrompt,
        style: style,
        format: format
    )
    return outcome.image
}
```

## Multi-concept generation

For ImagePlayground, StoryFox uses a multi-concept approach with progressive shedding:

<Info>
  Instead of sending one long text string, the system decomposes prompts into ranked concepts and sends them as separate `.text()` items. If generation fails, it drops the least important concept and retries.
</Info>

```swift
let decomposition = await PromptAnalysisEngine.decomposeIntoConcepts(prompt: prompt)
var conceptCount = decomposition.concepts.count

while conceptCount > 0 {
    let activeConcepts = Array(decomposition.concepts.prefix(conceptCount))
    
    do {
        let outcome = try await router.generateImage(
            prompt: prompt,
            style: style,
            format: format,
            rankedConcepts: activeConcepts
        )
        return outcome.image
    } catch {
        // Shed least important concept and retry
        conceptCount -= 1
    }
}
```

## Character prefix preservation

Character descriptions are preserved across all prompt variants:

```swift
private func splitCharacterPrefix(from prompt: String) -> (prefix: String, scene: String) {
    // New format: "Featuring Luna, a small orange fox with a green scarf. <scene>"
    if prompt.hasPrefix("Featuring ") {
        if let dotRange = prompt.range(of: ". ") {
            let prefix = String(prompt[..<dotRange.upperBound])
            let scene = String(prompt[dotRange.upperBound...])
            return (prefix, scene)
        }
    }
    return ("", prompt)
}
```

The prefix is kept intact while only the scene description is truncated in fallback variants. This ensures character consistency is never lost during retries.

## Retry logic

The system includes smart retry logic that distinguishes transient failures from content safety blocks:

```swift
private func shouldRetrySameVariant(after error: Error) -> Bool {
    guard let imageError = error as? ImageCreator.Error else {
        return true  // Unknown failures - retry once
    }
    
    switch imageError {
    case .unavailable:
        return true  // Transient system resource issue
    case .creationFailed:
        return false  // Content filtering - escalate to next variant
    default:
        return false
    }
}
```

## Recovery passes

If pages fail during the parallel generation phase, recovery passes retry them sequentially:

```swift
for pass in 1...GenerationConfig.imageRecoveryPasses {
    let startVariant = min(pass, 5)  // Skip variants that already failed
    
    for job in pending {
        let image = try await generateSingleImage(
            prompt: job.prompt,
            style: style,
            format: format,
            startingVariantIndex: startVariant
        )
        generatedImages[job.index] = image
    }
}
```
